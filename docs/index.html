nblex

Network & Buffer Log EXplorer

A high-performance, real-time observability platform that uniquely combines application log analysis with network traffic monitoring.

What is nblex?

nblex is a lightweight observability tool that correlates application logs with network traffic in real-time. Unlike traditional tools that analyze logs OR network traffic in isolation, nblex links events across both layers to provide unprecedented visibility into system behavior.

Key Differentiator: Real-time correlation between application-level insights (logs) and network-level reality (packets).

Getting Started

Installation

# Clone the repository git clone https://github.com/yourorg/nblex.git cd nblex # Build from source mkdir build && cd build cmake .. make # Install (optional) sudo make install

Basic Usage

# Monitor a log file and network interface nblex -i file:/var/log/app.log -i pcap:eth0 # Filter for ERROR logs with high HTTP status codes nblex -i file:/var/log/app.log -f "level == 'ERROR' AND status >= 500" # Output to file with rotation nblex -i file:/var/log/app.log -o file:/var/log/correlated.log

Configuration File

# Example config.yaml inputs: - name: app_logs type: file path: /var/log/application/*.log - name: network type: pcap interface: eth0 outputs: - name: console type: stdout - name: metrics type: prometheus path: /var/lib/prometheus/nblex.metrics correlation: enabled: true window_ms: 100

Architecture

nblex uses an agent-based architecture with three main components:

  • Input Agents: Collect data from logs, network interfaces, and other sources
  • Processing Pipeline: Parse, filter, and correlate events in real-time
  • Output Agents: Format and deliver processed events to various destinations

The correlation engine is nblex's key differentiator, automatically linking events across layers using configurable rules and time windows.

How nblex Compares

Unlike heavy-weight observability platforms, nblex is designed to be:

  • Lightweight: Minimal dependencies, low resource usage
  • Focused: Does one thing well - log/network correlation
  • Fast: Real-time processing with sub-millisecond latency
  • Simple: Easy to deploy and configure for development and production

nblex complements existing tools like Elasticsearch, Prometheus, and Jaeger rather than competing with them.
